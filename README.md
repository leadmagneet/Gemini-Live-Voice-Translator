# Gemini Live Voice Translator

> **Windows-only experimental real-time meeting translator** using Gemini Live Translate, VB-CABLE, VoiceMeeter, Node.js, and WASAPI audio routing.

Gemini Live Voice Translator is a local speech-to-speech translator for online meetings and calls.

It can translate your microphone into another language and send the translated voice into a meeting as a virtual microphone. It can also capture incoming meeting audio, translate it back, and play the translated voice into your headphones.

Example use case:

```txt
You speak Russian → Gemini translates to English → meeting receives English audio
Remote person speaks English → Gemini translates to Russian → you hear Russian audio
```

The language directions are configurable.

---

## Platform support

This project is currently designed for:

```txt
Windows 10 / Windows 11
```

It depends on Windows audio routing through:

- VB-CABLE
- VoiceMeeter
- WASAPI audio devices through `audify`

Linux and macOS are not officially supported yet. They may require different virtual audio routing and code changes.

---

## Important limitations

This is not a finished commercial product. It is a developer tool / experimental real-time translator.

Expected delay is usually around:

```txt
5–7 seconds
```

Actual latency depends on:

- microphone and audio driver latency
- Gemini API response time
- network connection
- meeting app audio processing
- VoiceMeeter / VB-CABLE routing
- playback buffer settings
- how long the speaker phrase is

The translated voice is generated by the model and may not preserve the original speaker identity. It can sometimes sound like a different person, change tone, or even sound like a different gender.

Real-time speech-to-speech translation can still have:

- latency
- recognition mistakes
- wrong translations
- missing words
- audio routing issues
- model interruptions
- voice changes
- Gemini API availability or quota problems

Do not use this for legal, medical, emergency, financial, or safety-critical interpretation.

## How it works

The app uses two separate audio routes.

### 1. Outgoing route

Your real microphone goes into Node.js. Gemini translates it. The translated audio is sent to VB-CABLE.

```txt
Physical microphone
→ Node.js
→ Gemini Live Translate
→ CABLE Input
→ CABLE Output
→ Meeting microphone
```

In your meeting app, select:

```txt
Microphone = CABLE Output
```

### 2. Incoming route

The meeting audio is sent to VoiceMeeter. Node.js captures `Voicemeeter Out B1`, sends it to Gemini, and plays the translated audio to your headphones.

```txt
Meeting speakers
→ Voicemeeter Input
→ Voicemeeter Out B1
→ Node.js
→ Gemini Live Translate
→ Real headphones
```

In your meeting app, select:

```txt
Speakers = Voicemeeter Input
```

In VoiceMeeter, set:

```txt
Voicemeeter Input:
A = OFF
B = ON
```

This sends meeting audio to the virtual B1 output without also playing the original audio directly into your headphones.

# Installation

Follow this order. Do not start `index.js` until all audio routing is configured.

---

## Step 1 — Download and install virtual audio drivers

This project requires two third-party VB-Audio tools:

- VB-CABLE
- VoiceMeeter

The installers are **not included** in this repository.

Download them only from the official VB-Audio websites:

```txt
VB-CABLE:
https://vb-audio.com/Cable/index.htm

VoiceMeeter:
https://vb-audio.com/Voicemeeter/
```

Install both tools as Administrator.

Recommended order:

1. Download VB-CABLE from the official website.
2. Install VB-CABLE as Administrator.
3. Download VoiceMeeter from the official website.
4. Install VoiceMeeter as Administrator.
5. Restart Windows.

Restarting Windows is important. Without a reboot, Windows audio devices may not appear correctly.

### About VB-Audio licensing

VB-CABLE and VoiceMeeter are third-party VB-Audio software.

VoiceMeeter is donationware. After 30 days, it may invite you to buy a license or continue evaluation. If you find it useful, please support VB-Audio.

This repository does not redistribute, modify, or bundle VB-Audio installers.

---

## Step 2 — Install Node.js

Install Node.js 18 or newer:

```txt
https://nodejs.org/
```

After installation, open Command Prompt or PowerShell and check:

```bash
node -v
npm -v
```

If both commands print versions, Node.js and npm are installed.

---

## Step 3 — Clone or download this repository

Using Git:

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
cd YOUR_REPO_NAME
```

Or download the repository as ZIP from GitHub, extract it, and open the extracted folder in Command Prompt or PowerShell.

---

## Step 4 — Install Node.js dependencies

Run:

```bash
npm install
```

This installs the required Node.js packages into the local project folder:

```txt
node_modules/
```

It does not install them globally.

The project uses these npm packages:

| Package | Purpose |
|---|---|
| `ws` | WebSocket connection to Gemini Live API |
| `dotenv` | Loads configuration from `.env` |
| `audify` | Captures and plays raw PCM audio through WASAPI |

---

## Step 5 — Create `.env`

Create a local `.env` file:

```bash
copy .env.example .env
```

Do not commit `.env` to GitHub because it contains your private Gemini API key and local device IDs.

---

## Step 6 — Open the setup window

Run:

```bash
npm run config
```

or:

```bash
node config-window.js
```

The setup window should open automatically in your browser:

```txt
http://localhost:3000
```

If it does not open automatically, copy the URL from the terminal and open it manually.

The setup window is local. It uses Node.js built-in HTTP server. It does not require Express or Electron.

---

## Step 7 — Configure and test devices

In the setup window:

1. Click **Refresh devices**.
2. Click **Auto-fill detected**.
3. Paste your Gemini API key.
4. Click **Test Gemini key**.
5. Select your physical microphone.
6. Select `CABLE Input` as the outgoing translated audio device.
7. Select `Voicemeeter Out B1` as the incoming meeting audio device.
8. Select your headphones or speakers as translated playback output.
9. Run the RMS and beep tests.
10. Click **Save .env**.

Recommended device mapping:

```txt
REAL_MIC_ID        = your physical microphone
CABLE_INPUT_ID     = CABLE Input
VOICEMEETER_OUT_ID = Voicemeeter Out B1
REAL_PHONES_ID     = your headphones / speakers
```

Do not copy device IDs from another computer. Device IDs are local to each Windows installation.

---

## Step 8 — Configure your meeting app

In Zoom, Google Meet, Discord, Microsoft Teams, or another meeting app, select:

```txt
Microphone = CABLE Output
Speakers   = Voicemeeter Input
```

Do **not** select `CABLE Input` as the meeting microphone.

Do **not** select your real headphones as the meeting speakers if you want incoming translation to work through VoiceMeeter.

---

## Step 9 — Configure VoiceMeeter

Open VoiceMeeter.

Find the `Voicemeeter Input` strip.

Set:

```txt
A = OFF
B = ON
```

Recommended:

```txt
Hardware Out A1 = your real headphones
Voicemeeter Input A button = OFF
Voicemeeter Input B button = ON
```

This means:

```txt
Meeting audio goes to B1
Node.js captures B1
Gemini translates it
Node.js plays translated audio to your real headphones
```

---

## Step 10 — Run the translator

After `.env` is saved and your meeting app is routed correctly, stop the setup window server with:

```bash
Ctrl + C
```

Then run:

```bash
npm start
```

or:

```bash
node index.js
```

Expected logs:

```txt
[SYSTEM] OUTGOING_TARGET_LANG=en
[SYSTEM] INCOMING_TARGET_LANG=ru
[WS OUTGOING_TO_EN] Connected
[WS INCOMING_TO_RU] Connected
```

When you speak:

```txt
[OUTGOING_TO_EN INPUT] ...
[OUTGOING_TO_EN OUTPUT] ...
```

When the remote person speaks:

```txt
[INCOMING_TO_RU INPUT] ...
[INCOMING_TO_RU OUTPUT] ...
```

# Usage

## Normal usage flow

1. Start your meeting app.
2. Set meeting microphone to `CABLE Output`.
3. Set meeting speakers to `Voicemeeter Input`.
4. Open VoiceMeeter and set `Voicemeeter Input: A = OFF, B = ON`.
5. Run the translator:

```bash
npm start
```

6. Speak into your microphone.
7. The meeting receives translated audio.
8. When remote participants speak, you hear translated audio in your headphones.

---

## Opening official driver download pages

If Node.js is already installed and dependencies are installed, you can open the official VB-Audio download pages with:

```bash
npm run downloads
```

This only opens the official websites. It does not download or install anything automatically.

## Setup window

Run:

```bash
npm run config
```

The setup window helps you configure the project without manually editing `.env`.

It can:

- list Windows audio devices
- detect `CABLE Input`
- detect `CABLE Output`
- detect `Voicemeeter Input`
- detect `Voicemeeter Out B1`
- select your physical microphone
- select translated output device
- test Gemini API key
- send a beep to a selected output device
- measure RMS from a selected input device
- save `.env`

# Troubleshooting

## Nothing works after changing `.env`

Stop all running Node processes:

```bash
Ctrl + C
```

Then run setup again:

```bash
npm run config
```

Check devices again and save `.env`.

---

## Gemini key fails

Check that:

- the API key is correct
- the Gemini API is enabled for your account/project
- the selected model is available
- your internet connection works
- you have not exceeded quota or rate limits

---

## Outgoing translation works, incoming does not

Check meeting settings:

```txt
Speakers = Voicemeeter Input
```

Check VoiceMeeter:

```txt
Voicemeeter Input:
A = OFF
B = ON
```

Check `.env`:

```env
VOICEMEETER_OUT_ID=ID_of_Voicemeeter_Out_B1
```

Run the incoming RMS test in the setup window. Speak from another device in the meeting. RMS should rise.

---

## Incoming translation works, outgoing does not

Check meeting settings:

```txt
Microphone = CABLE Output
```

Check `.env`:

```env
CABLE_INPUT_ID=ID_of_CABLE_Input
REAL_MIC_ID=ID_of_your_physical_microphone
```

Run the beep test for `CABLE_INPUT_ID`. The meeting microphone meter should move.

---

## I hear the original meeting audio and the translated audio

In VoiceMeeter, turn off `A` on the `Voicemeeter Input` strip:

```txt
A = OFF
B = ON
```

If `A` is enabled, the original meeting audio can go directly to your headphones.

---

## The translator hears itself

This usually means the routing is wrong.

Check:

```txt
Meeting Microphone = CABLE Output
Meeting Speakers   = Voicemeeter Input
```

Do not route Gemini output back into its own input.

---

## Audio has too much delay

The normal delay is about:

```txt
5–7 seconds
```

You can try lowering:

```env
MAX_PLAYBACK_QUEUE_MS=700
```

The default is safer:

```env
MAX_PLAYBACK_QUEUE_MS=900
```

Very low values can cause audio drops.

---

## First words are missing

Keep the stable configuration:

```env
REALTIME_INPUT_CONFIG=0
GEMINI_CHUNK_MS=100
CAPTURE_FRAMES=800
```
